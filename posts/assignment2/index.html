I. Deployment Process Overview

1. Local Blog Setup with Hugo

Step 1: Install Tools

  • Visual Studio Code: Text editor for editing toml, yaml, and md files.
  • Git Bash / PowerShell: For running commands and managing git repositories.
  • Hugo Extended: Required for building themes that use SCSS.

Install Hugo using scoop on Windows:

scoop install hugo-extended

Step 2: Create Hugo Blog

Navigate to your desired directory and create the blog:

cd D:/Kexin/blog
hugo new site myblog
cd myblog
git init

Step 3: Add a Theme

Clone the Hugo theme (e.g., Dream) into the themes directory:

git submodule add https://github.com/cycinter/hugo-theme-dream.git themes/dream

Modify hugo.toml in the root directory to include theme configuration (based on the theme’s example file).

Step 4: Launch Local Server

Start Hugoâ€™s development server to preview the site locally:

hugo server -D

Visit http://localhost:1313/ in your browser to view the blog.

2. Publishing Blog Posts

Create a new post using Hugo’s CLI:

hugo new content/posts/my-first-post.md

Edit the generated Markdown file to add content. Set draft = false in the front matter to make it publicly visible.

II. GitHub Deployment with Dual Repositories

Step 1: Create GitHub Repositories

  • Repository 1 (source): Contains all the Hugo source files.
  • Repository 2 (pages): Must be named yourusername.github.io and used for GitHub Pages deployment.

Create a Personal Access Token (PAT) on GitHub with permissions: repo, workflow, admin:repo_hook.

Step 2: Configure GitHub Actions Workflow

Create .github/workflows/hugo.yaml in your project root:

name: GitHub Pages

on:
  push:
    branches:
      - main

jobs:
  deploy:
    runs-on: ubuntu-20.04
    steps:
      - uses: actions/checkout@v3
        with:
          submodules: true
          fetch-depth: 0

      - name: Setup Hugo
        uses: peaceiris/actions-hugo@v2
        with:
          hugo-version: 0.139.4
          extended: true

      - name: Build
        run: hugo --minify

      - name: Deploy
        uses: peaceiris/actions-gh-pages@v3
        with:
          personal_token: ${{ secrets.MYTOKEN }}
          external_repository: yourusername/yourusername.github.io
          publish_dir: ./public
          publish_branch: main

Step 3: Push to GitHub

Set main branch, add remotes, and push:

git branch -M main
git remote add source https://github.com/yourusername/myblog.git
git remote add pages https://github.com/yourusername/yourusername.github.io.git
git add .
git commit -m "Initial commit with Hugo site"
git push source main

The GitHub Actions workflow will automatically build and deploy your blog.
